# PMS 系统自动签章助手

这是一个用于 PMS 电子签章流程的 Tampermonkey 用户脚本，主要帮助用户自动完成待办进入、批量签章、文件切换、签章确认等重复操作。

## 当前测试版本

当前分支版本：`v1.1.9-beta`

这是测试版本，用于验证签章系统页面变化后的流程稳定性。普通用户稳定使用建议保留 `v1.1.8`，确认测试稳定后再升级到正式版。

版本号以 `autoSign.js` 文件头部为准：

```javascript
// @version      1.1.9-beta
```

## 本次 beta 重点

- 待办页找不到“电子签章”入口时，自动判断为没有电子签章待办，并停止运行。
- 优化签字页切换下一个文件的等待逻辑，减少因为页面选中状态刷新慢导致的误判。
- 新增“文件加载超时”设置，默认 5 分钟，网络慢或文件较大时可以适当调大。
- 签章位置改为九宫格位置选择，并支持从 9 个固定位置中随机选择。
- 优化设置面板布局，让查询间隔、文件加载超时等设置更规整。
- 增强本轮已签文件的本地标记，减少页面状态刷新慢时重复选择刚签过文件的情况。

## 适合谁测试

- 需要验证签章系统升级后是否还能连续签章的用户。
- 遇到过“切换到下一个文件后卡住”的用户。
- 愿意在小范围内测试并反馈日志的用户。

如果用于正式工作，建议先保留 `v1.1.8` 作为回退版本。

## 安装方式

测试版建议从 GitHub Release 下载 `v1.1.9-beta` 附件后手动安装，不建议直接覆盖给所有用户。

1. 安装 Edge 浏览器和 Tampermonkey，也叫“篡改猴”。
2. 下载 `autosign-v1.1.9-beta.js`。
3. 打开 Tampermonkey 管理面板。
4. 新建脚本，粘贴测试版脚本内容并保存。
5. 打开 PMS 相关页面，确认左上角出现“运行”“设置”等按钮。

如果你通过 GreasyFork 安装正式版，建议先不要让普通用户同步 beta。测试稳定后再发布正式版本。

## 基本使用

1. 打开 PMS 待办页面或签章相关页面。
2. 点击左上角“运行”。
3. 脚本会自动查找电子签章入口，并进入后续流程。
4. 运行中可以通过状态提示查看当前步骤。
5. 如需暂停，点击“停止运行”。

常见状态包括：查找入口、等待页面加载、刷新列表、勾选文件、切换文件、等待签名模块、确认签章等。

## 设置项

### 签章位置

可以选择签章落在页面的 9 个常用位置：

```text
左上   上中   右上
左中   居中   右中
左下   下中   右下
```

选择“随机”时，脚本会在这 9 个固定位置中随机选择一个。

### 查询间隔

默认：`3 秒`

用于后续批次刷新待签章列表时控制查询节奏。网络或电脑较慢时，可以适当调大，让列表状态更稳定后再继续签章。

### 文件加载超时

默认：`5 分钟`

用于切换到下一个待签文件后，等待页面、画布和签名模块加载完成。网络较慢或文件较大时，可以适当调大。

## 常见问题

### 页面没有出现运行按钮

请检查：

- Tampermonkey 是否已启用。
- 脚本是否已安装并启用。
- 当前页面是否允许 Tampermonkey 运行用户脚本。
- 浏览器右上角 Tampermonkey 图标中，该脚本是否处于启用状态。

### 第一次使用卡在批量签章勾选页面

大概率是浏览器弹窗权限没有开启。

处理方式：

1. 查看浏览器地址栏右侧是否有“弹窗被拦截”图标。
2. 点击该图标。
3. 选择允许当前网站弹出窗口。
4. 刷新页面后重新运行脚本。

### 页面长时间没有继续

可以先观察左上角状态提示，确认脚本卡在哪一步。常见原因包括网络慢、页面加载慢、弹窗权限未开启、签章系统页面结构调整等。

如果无法判断，请在设置中点击“导出日志”，结合当时看到的页面情况一起反馈。
