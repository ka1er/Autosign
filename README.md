# PMS 系统自动签章助手

这是一个用于 PMS 电子签章流程的 Tampermonkey 用户脚本，主要帮助用户自动完成待办进入、批量签章、文件切换、签章确认等重复操作。

![Autosign v1.1.9 使用说明](https://github.com/ka1er/Autosign/releases/download/v1.1.9/autosign-v1.1.9-guide.png)

## 当前推荐版本

当前稳定版本：`v1.1.9`

`v1.1.9` 在 `v1.1.8` 的基础上继续优化签章系统升级后的流程稳定性，重点减少待办页误运行、签字页切换文件卡住、重复选择刚签过文件等问题。

版本号以 `autoSign.js` 文件头部为准：

```javascript
// @version      1.1.9
```

## 适合谁使用

- 经常需要在 PMS 中处理电子签章待办的用户。
- 需要连续处理多份签章文件的用户。
- 希望减少重复点击、减少等待和漏操作的用户。

脚本会尽量等待页面加载完成后再继续操作，但电子签章页面本身受网络、浏览器权限和系统页面变化影响。如果遇到异常，建议先导出日志再反馈。

## 安装方式

推荐使用 Edge 浏览器和 Tampermonkey。

1. 打开 Edge 浏览器。
2. 在 Edge 扩展商店安装 Tampermonkey，也叫“篡改猴”。
3. 安装完成后，确认浏览器右上角可以看到 Tampermonkey 图标。
4. 安装或复制本项目中的 `autoSign.js` 脚本。
5. 打开 PMS 相关页面，确认左上角出现“运行”“设置”等按钮。

如果通过 GreasyFork 安装，可以在 Tampermonkey 中使用脚本更新功能获取正式版本更新。

## 基本使用

1. 打开 PMS 待办页面或签章相关页面。
2. 点击左上角“运行”。
3. 脚本会自动查找电子签章入口，并进入后续流程。
4. 运行中可以通过状态提示查看当前步骤。
5. 如需暂停，点击“停止运行”。

常见状态包括：查找入口、等待页面加载、刷新列表、勾选文件、切换文件、等待签名模块、确认签章等。

## 设置项

### 签章位置

可以选择签章落在页面的 9 个常用位置：

```text
左上   上中   右上
左中   居中   右中
左下   下中   右下
```

选择“随机”时，脚本会在这 9 个固定位置中随机选择一个，而不是随意落在页面任意位置。

### 查询间隔

默认：`3 秒`

用于后续批次刷新待签章列表时控制查询节奏。网络或电脑较慢时，可以适当调大，让列表状态更稳定后再继续签章。

### 文件加载超时

默认：`5 分钟`

用于切换到下一个待签文件后，等待页面、画布和签名模块加载完成。网络较慢或文件较大时，可以适当调大。

## 常见问题

### 页面没有出现运行按钮

请检查：

- Tampermonkey 是否已启用。
- 脚本是否已安装并启用。
- 当前页面是否允许 Tampermonkey 运行用户脚本。
- 浏览器右上角 Tampermonkey 图标中，该脚本是否处于启用状态。

### 第一次使用卡在批量签章勾选页面

大概率是浏览器弹窗权限没有开启。

处理方式：

1. 查看浏览器地址栏右侧是否有“弹窗被拦截”图标。
2. 点击该图标。
3. 选择允许当前网站弹出窗口。
4. 刷新页面后重新运行脚本。

### 页面长时间没有继续

可以先观察左上角状态提示，确认脚本卡在哪一步。常见原因包括网络慢、页面加载慢、弹窗权限未开启、签章系统页面结构调整等。

如果无法判断，请在设置中点击“导出日志”，结合当时看到的页面情况一起反馈。

## 版本变化

### v1.1.9

- 待办页找不到电子签章入口时，也会按“没有电子签章待办”处理并停止运行。
- 优化签字页切换下一个文件的等待逻辑，减少因为页面选中状态刷新慢导致的误判。
- 新增文件加载超时设置，默认 5 分钟，网络慢或文件较大时可以适当调大。
- 签章位置改为九宫格位置选择，并支持从 9 个固定位置中随机选择。
- 优化设置面板布局，让查询间隔、文件加载超时等设置更规整。
- 增强本轮已签文件的本地标记，减少页面状态刷新慢时重复选择刚签过文件的情况。

### v1.1.8

- 优化批量签章列表刷新逻辑，减少重复处理同一批文件的情况。
- 优化签字页文件切换，等待页面、画布和签名模块准备好后再继续。
- 新增查询间隔设置，网络较慢时可以手动调大。
- 新增日志导出功能，方便反馈问题。
